# 教材游戏分析报告 —— Pong 游戏

> 教材：《现代C++编程实战》第4章实战项目之二：Pong游戏
> 文件：`4_pong.cpp`

---

## 一、游戏概述

Pong 是一个经典的双人对战弹球游戏。画面中间有一个球自动运动，左右两侧各有一块挡板分别由两位玩家控制（W/S 键控制左挡板，方向键↑↓控制右挡板），球碰到挡板反弹并得分，球出界则对方得分，按 Q 退出。

---

## 二、控制语句分析

### 1. `while` —— 游戏主循环（第 100 行）

```cpp
while (true) { ... }
```

实现游戏的核心运行逻辑：**无限循环**驱动每一帧的画面更新。每一轮迭代依次执行键盘事件处理 → 数据更新 → 画面绘制 → 延时 50ms，形成流畅的实时游戏体验。

### 2. `if-else if-else` —— 键盘输入分支（第 107–123 行）

```cpp
if ((key == 'w' || key == 'W') && paddle1_y > paddle1_vec) { ... }
else if ((key == 's' || key == 'S') && ...) { ... }
else if (key == 72 && ...) { ... }
else if (key == 80 && ...) { ... }
else if (key == 'q' || key == 'Q') { break; }
```

根据玩家按键执行不同操作：W/S 移动左挡板、方向键移动右挡板、Q 退出游戏。每个分支还附加了边界条件（挡板不能超出画面），展示了 `if-else if` 的完整用法。

### 3. `if` —— 碰撞检测与计分（第 130–153 行）

```cpp
if (ball_y < 0 || ball_y >= HEIGHT) { ball_vec_y = -ball_vec_y; }        // 上下墙反弹
if (ball_x < paddle_w && ...) { ball_vec_x = -ball_vec_x; score1 += 1; }  // 碰左挡板
else if (ball_x > WIDTH - paddle_w && ...) { ... }                         // 碰右挡板
if (ball_x < 0 || ball_x > WIDTH) { ... }                                  // 出界得分
```

用独立的 `if` 判断球与墙壁、挡板、边界的碰撞，实现物理逻辑和计分规则。这里用 `if-else if` 区分碰左/右挡板（互斥事件），而其他碰撞检测用独立 `if`（可能同时发生）。

### 4. `for` —— 帧缓冲绘制（第 170–205 行）

```cpp
for (auto x = 0; x < WIDTH; x++) { framebuffer[frame_idx++] = '='; }  // 顶部墙
for (auto y = 1; ...; y++) {          // 逐行遍历
    for (auto x = 0; ...; x++) {      // 逐列遍历
        if (...) ...                   // 判断每个像素该画什么
    }
}
for (auto x = 0; x < WIDTH; x++) { ... }  // 底部墙
```

用嵌套 `for` 循环逐行逐列填充画面缓冲区，实现整屏画面的批量渲染。外层循环遍历行，内层循环遍历列，每格根据坐标判断是球(O)、挡板(Z)、墙壁(=)还是空白。

### 5. `break` —— 退出游戏（第 122 行）

```cpp
else if (key == 'q' || key == 'Q') { break; }
```

在按键处理分支中，按 Q 触发 `break` 跳出 `while(true)` 主循环，结束游戏。

### 6. `#ifdef / #else / #endif` —— 平台适配（多处）

预处理指令用于区分 Windows 和 Linux/macOS 平台：`_kbhit()`/`_getch()` 的非阻塞键盘读取、光标的隐藏与显示、控制台光标定位。虽然不是运行时控制语句，但体现了条件编译在跨平台开发中的典型应用。

## 三、控制语句使用汇总

| 控制语句 | 出现位置 | 实现功能 |
|---------|---------|---------|
| `while` | 第 100 行 | 游戏主循环，每帧执行事件→更新→渲染 |
| `if-else if-else` | 第 107–123 行 | 键盘输入分发（移动挡板 / 退出） |
| `if` / `else if` | 第 130–153 行 | 碰撞检测与物理反弹 / 计分 |
| `for`（含嵌套） | 第 170–205 行 | 双层循环遍历像素，绘制整屏画面 |
| `break` | 第 122 行 | 退出游戏主循环 |
| `#ifdef / #else` | 多处 | 跨平台条件编译 |

## 五、教材游戏与我设计的游戏的对比

| 维度 | 教材 Pong 游戏 | 我的海岛求生 |
|------|---------------|-------------|
| 游戏类型 | 实时动作（双人对战） | 回合策略（单人求生） |
| 交互方式 | 非阻塞键盘（_kbhit） | 阻塞式菜单选择（cin） |
| `switch` | 未使用 | 白天行动菜单（5 case + default） |
| `do-while` | 未使用 | 输入验证（行动选择 + 重玩询问） |
| `continue` | 未使用 | 死亡后跳过夜间阶段 |
| `for` | 帧缓冲区嵌套绘制 | 文本进度条 + 倒计时动画 |
| 主循环 | `while(true)` + `break` 退出 | 双层 `while`（日循环 + 重玩循环） |
| 输入容错 | 无（仅响应有效按键） | 有（cin.fail() 检测 + 范围校验） |
